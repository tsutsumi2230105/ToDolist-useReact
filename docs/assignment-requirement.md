# Week 3-4 課題：React + TypeScript TODO アプリケーション

## 課題概要

React と TypeScript を使用して、TodoMVC をベンチマークとした TODO リストアプリケーションを作成してください。

**ベンチマークサイト：** https://todomvc.com/examples/javascript-es6/dist/

## 学習目標

- React の基本概念（コンポーネント、State、Props）の理解
- TypeScript の型定義を活用した開発
- localStorage を用いたデータ永続化
- ユーザーインターフェースの実装
- Git ブランチ運用の習得
- Sass を用いたスタイリング

## 必須要件

### 機能要件

1. **TODO の追加**

   - 入力フォームから TODO を追加できること
   - 空の入力は追加できないようにすること

2. **TODO の表示**

   - 追加した TODO が一覧表示されること
   - 各 TODO に完了/未完了のチェックボックスがあること

3. **TODO の編集**

   - TODO をダブルクリックで編集モードに切り替え
   - 編集内容を保存できること

4. **TODO の削除**

   - 各 TODO に削除ボタンがあること
   - クリックで削除できること

5. **フィルター機能**

   - All（全て）
   - Active（未完了のみ）
   - Completed（完了済みのみ）

6. **一括操作**

   - 全ての TODO を完了/未完了にするトグル機能
   - 完了済み TODO を一括削除する機能

7. **カウンター表示**

   - 未完了の TODO 数を表示

8. **データ永続化**
   - localStorage を使用してデータを保存
   - ページリロード後もデータが保持されること

### 技術要件

- **React** (v18 以上推奨)
- **TypeScript**
- **Sass (SCSS)** によるスタイリング
- **localStorage** によるデータ管理
- Create React App または Vite での環境構築

### Git 運用要件（必須）

以下のブランチ運用ルールを **必ず** 守ってください。

1. **ブランチ戦略**

   - `main` 本番相当の安定版
   - `feat/*` 新機能の追加
   - `fix/*` バグ修正
   - `style/*` スタイリングの変更
   - `refactor/*` コードのリファクタリング
   - `docs/*` ドキュメントの変更
   - `chore/*` ビルドプロセスやツールの変更

2. **開発フロー**

```
   main ← feat/機能名
```

3. **ブランチ命名規則**

   - 機能開発：`feat/add-todo-input`
   - 機能開発：`feat/filter-function`
   - 機能開発：`style/common-button-layout`
   - など、わかりやすい名前をつける

4. **作業手順**

   1. `main` ブランチから `feat/*` ブランチを作成
   2. 機能を実装
   3. コミットメッセージは具体的に記述
   4. `main`に向けて PR
   5. レビューを受け approve を受けたら`main` ブランチにマージ
